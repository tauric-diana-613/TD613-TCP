## 2. Eight custody questions

The entrant can now recover these questions before acting:

1. **What is preserved** — current local case identity, exact Save Point continuity snapshot, attached local receipts, and visible digest posture.
2. **What remains outside** — unselected source material, external systems, recipient-side state, unstored passphrases, unresolved questions, and unsupported claims.
3. **Who may open it** — a human possessing the encrypted copy and passphrase; Ash stores no passphrase and infers no opener.
4. **What closes it** — a deliberate human Save Point or encrypted export action through the existing continuity engine.
5. **Where it may go** — encrypted export may prepare a copy; destination crossing remains separately authorized.
6. **Which receipt follows it** — Save Point, export, release, route, recipient, and handoff receipts remain non-equivalent.
7. **How it may return** — encrypted import may open an authenticated copy without silently overwriting the current Case Map.
8. **What sealing does not prove** — truth, authorship, identity, intent, causation, consent, safety, completeness, external custody, recipient deletion, or permission to release.

## 3. Save Point recompilation

A11 presents Save Points as continuity snapshots and preserves four distinctions:

```text
preservation
≠ sealing
≠ save state

```

A Save Point supplies a return address and local digest posture. It does not rewind the present, overwrite the current Case Map, erase open questions, or infer that the earlier state was true or complete.
