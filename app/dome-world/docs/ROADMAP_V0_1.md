# TD613 Aperture / Flow-Core / Dome-World / Ash Roadmap v0.1

**Working title:** *The Crayon Map*  
**Status:** governed implementation roadmap  
**Baseline:** Aperture v3.0-alpha · Dome-World v0.5.0 · Ash v0.7 commitment / v0.6 derivative lineage · exact substrate v0.4.3

## Governing laws

- **Open Field governs idea generation; review begins only at operator-requested promotion.**
- **Artifact, context, and relation are separate strata.**
- **Receipts may cross stations; authority does not.**
- **The human custodian remains the decision point for consequential Ash actions.**
- **Proof is limited to propositions whose truth conditions formally close.**
- **Prediction remains research-only until calibration, abstention, drift, and harm tests are satisfied.**
- **This roadmap introduces no new claim-ceiling mechanism. Existing legacy vocabulary remains frozen for separate review.**

## Phase 0 — Baseline Freeze and Contract Ledger

**Purpose:** Freeze the stable instrument, inventory every endpoint and schema, preserve plural version lineage, and establish rollback and regression receipts before semantic expansion.

**Exit gate:** endpoint inventory complete; current routes documented; inherited modules cannot rewrite current identity; deployment probe exists; no phase begins from undocumented behavior.

## Phase 1 — Ash Local Commitment Kernel

**Purpose:** Compute a true browser-local SHA-256 over exact selected bytes and register either `L0_METADATA_ONLY` or `L1_BROWSER_LOCAL_ARTIFACT_DIGEST` without transmitting raw bytes.

**Exit gate:** known vectors, empty file, mutation, Unicode byte divergence, finite file-size hold, no placeholder digest, no metadata-derived artifact digest, strict L1 boundary validation, replay without content, stale-selection race blocked.

## Phase 2 — Canonical Digest and Receipt Spine

**Purpose:** Separate `artifact_digest`, `manifest_digest`, and `receipt_digest`; define canonical JSON and migration rules without turning any digest into identity, authorship, possession, authenticity, or time proof.

## Phase 3 — Flow-Core Context Instrumentation

**Purpose:** Add named sensors, source status, transformation history, missingness, uncertainty, alternatives, benign controls, and abstention. Context remains artifact-blind and private by default.

## Phase 4 — Reciprocal Receipt Bridge

```text
Aperture diagnostic receipt
→ Flow-Core contextual translation
→ Flow-Core context receipt
→ Aperture returned-receipt audit
→ recommendation, not command
→ optional Ash human gate
```

**Purpose:** Implement two-way communication before two-way authority. Flow-Core cannot write Aperture doctrine; Aperture cannot execute Ash.

## Phase 5 — Relation Envelope and Phason Continuity

**Purpose:** Permit an operator-approved, local-only relation between an Ash reference and Flow-Core context receipt without exposing a universal stable artifact digest. Phason records creation, revision, withdrawal, and supersession without claiming co-occurrence, identity, location, or causation.

## Phase 6 — Human-Gated Ash Derivatives

**Purpose:** Redesign Veil, Cinder, Compare, Recall, and export around explicit confirmation, separate authorization/execution receipts, client-side or destination-bound fragment handling, and honest non-erasure boundaries.

**Hard stop now:** plaintext Cinder transport remains held until this phase.

## Phase 7 — Independent Provenance Adapters

**Purpose:** Add optional signature, RFC 3161, Git anchor, Merkle inclusion, and independent witness adapters. Each adapter states its own trust assumptions and proves only its narrow proposition.

## Phase 8 — Advanced Privacy Research Lane

**Purpose:** Keep ZK, differential privacy, and threshold-access work in Open Field until predicates, parsers, neighboring datasets, sensitivity, auxiliary-information assumptions, and failure modes are explicit.

## Phase 9 — Validation, Deployment, and Release Discipline

**Purpose:** Require phase-specific CI, browser/mobile parity, live probes, schema compatibility receipts, security review, rollback paths, and a release receipt that separates operational code from research surfaces.
