## Ruling

The Ash Keep buildout succeeded as a serious browser-local custody and controlled-disclosure instrument. It now preserves Case Maps, Rooms, Route Memory, Rebuild receipts, Choir assays, exact local approvals, encrypted continuity, and bounded destination handoff without granting universal transport, prediction, deletion proof, suppression, or Cinder authority.

The buildout cannot support the stronger proposition that a complete whistleblowing investigation may enter a cloud AI terminal and become unreconstructable by every platform operator, workplace administrator, investigation subject, elite-access service, or future Reader. No receipt, animation, cryptographic digest, or named-Reader assay can make that proposition true after plaintext crosses into infrastructure outside the Keep.

The product law therefore remains:

> Never upload a full case because a receipt cannot un-send it.

## Sedgwickian devastation

The paranoid reading asks what the interface lets a hurried investigator believe before asking what the code technically promises.

Three seams required exposure:

1. **Local-looking terminal collapse.** A desktop app, browser tab, or shell may look local while transmitting plaintext to a provider.
2. **Destination-class collapse.** Consumer cloud AI, employer/public-sector managed AI, and a fully offline model were taught as distinct environments, yet the provider button originally gave those distinctions no executable jurisdiction.
3. **Browser-local overreach.** IndexedDB custody remains subject to the device, browser profile, extensions, origin code, backups, screenshots, workplace administration, and local human access. Capsule encryption protects exported continuity, not every live workspace state against a compromised endpoint.

The reparative reading preserves what Ash genuinely gives the operator: separation, minimization, declared-route testing, visible uncertainty, exact local approval, continuity, and the ability to stop.

### Fully offline local model

The provider action hard-holds through route mismatch. **Draft with Hush** invokes a configured external provider; it cannot impersonate an offline, network-isolated model merely because the user launched it from a local shell.


A Capsule is authenticated encrypted continuity. It should never be fed to an AI as though it were an analysis packet.

A provider packet is a minimized plaintext projection that deliberately leaves the Keep.

The heterostratigraphic lane may render the directional narrowing from dense local structure to a bounded projection. Flow-Core may receive tested vector summaries and receipt references. It remains artifact-blind and cannot enforce custody. Ash Court holds the executable crossing; Hush holds local screening and the configured provider route.

## Residual security work before high-risk reliance

The following remain outside PR #390 and should be treated as blockers for any claim stronger than browser-local controlled disclosure:

1. A hardened offline distribution whose code and assets can be pinned and verified before opening sensitive IndexedDB state.
2. Content Security Policy and related response headers for the Ash surface, with inline code progressively removed so `unsafe-inline` can disappear.
3. A documented extension-free, non-managed browser profile posture and an explicit managed-device hard stop at launch.
4. Optional encryption-at-rest for live case state, with clear key-loss and recovery law; Capsule encryption alone covers exported continuity.
5. A distinct offline-model handoff that never reuses the external Hush provider button or its semantics.
6. Adversarial tests for semantic completeness, chronology density, identity-table patterns, and stable joining keys in plain text, while preserving the claim that heuristics cannot certify universal non-reconstruction.
7. A local origin-integrity witness that shows the exact application build governing the IndexedDB opening event.
