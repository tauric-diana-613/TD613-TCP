## Ruling

The earlier Political Campaign and Fundraiser demos demonstrate local hydration, Room separation, Route Memory, draft defaults, and continuity prompts. They remain useful product demonstrations. They do not yet model the full evidentiary burden of a serious research workspace.

Their principal ceiling is not object count. It is experimental constitution. A dense Case Map may still remain scientifically thin when it lacks explicit preregistration, calibration, matched controls, held-outs, nulls, contradictory observations, alternative models, metadata recovery, joining-key pressure, ethics, environment qualification, and claim ceilings.

The Research profile therefore uses a stricter class:

```text
BASELINE_HYDRATION
≠
QUALIFICATION_HYDRATION
```

## Ingress membrane repair

The prior membrane used a fixed viewport and vertical centering without a governed scroll axis. Short desktop viewports could center a panel taller than the available block size, leaving controls clipped above or below the visible field.

The repaired membrane is a flex scroll container:

```text
fixed viewport
+ overflow-y:auto
+ auto panel margins
=
center when the panel fits
top-safe scroll when it does not
```

The membrane owns the scrollbar. The underlying case page remains fixed beneath it.

The multi-engine flight tests:

- Chromium;
- Firefox;
- WebKit;
- 1920×1080;
- 1536×864;
- 1366×768;
- 1440×700;
- 1280×600;
- 1024×768.

For fitting viewports, the panel must remain centered. For overflowing viewports, the action controls must become reachable at the bottom of the membrane’s own scroll range.

## Cache ruling

The earlier release already used `no-store` for the Ash document and selected scripts. That did not guarantee one coherent module generation across every browser cache, Cache Storage entry, service worker registration, back-forward restoration, or stale submodule URL.

The new epoch performs a one-time maintenance action:

- delete same-origin Cache Storage entries;
- unregister Dome/Ash service workers when present;
- request revalidation of the declared critical module graph using `cache: reload`;
- append a release epoch to the document URL;
- version-bind lifecycle, workspace bridge, profile hydration, premium UI, and Research hydration imports;
- retain a session-scoped cache-flush receipt.

It preserves:

- IndexedDB cases;
- Room Rules;
- Route Memory;
- receipts;
- Capsules;
- the current-case pointer.

It refuses the claim:

```text
JavaScript cache maintenance
=
universal browser HTTP-cache purge
```

The accurate claim is:

> Cache Storage, relevant service workers, and the declared critical asset graph were refreshed under the named release epoch; universal eviction of every browser, intermediary, and provider cache remains outside the instrument’s proof.
