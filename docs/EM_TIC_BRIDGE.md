# EM / TIC / JITAI BRIDGE

TCP overlaps with trauma-informed adaptive systems because both care about:

- state detection,
- timing,
- burden,
- and route selection under fragile conditions.

Where TCP adds something distinctive is in treating the user's strange or excessive residue as possible **custodial remainder** rather than mere distress signal.

That matters even more in the current Generator V2 build. A miss is no longer supposed to disappear into a polite near-source output. TCP now exposes a hold docket when the writer cannot land safely, which is closer to trauma-informed practice than a system that quietly overwrites difficulty with false smoothness.

## Translation table

| TCP term | TIC/JITAI adjacent term |
|---|---|
| route pressure | intervention timing pressure |
| mirror.off | low-cognitive-load safety mode |
| witness buffer | stabilization layer |
| protected core | high-integrity clinical/custodial layer |
| provenance retention | continuity / context retention |
| reuse gain | burden prevented by structured intervention reuse |

## Why this matters

A JITAI system usually asks: what support, when?  
TCP adds: what continuity work has already been externalized onto the person, and can the system lower it without erasing provenance?

In the current repo that also means: if the writer cannot carry the requested transform without breaking anchors or meaning, the correct answer is an explicit visible hold, not a silent substitution that makes the burden harder to audit.

That same rule now extends to the main TCP surface: the operator-facing readout should stay short and legible, while the full burden / warning / Aperture trace remains available in secondary detail for people who need it.
