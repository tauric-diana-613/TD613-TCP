# Hush Mobile Field Deck Audit

## Purpose

Mobile Hush should behave like a handheld field deck rather than a shrunken desktop cockpit.

## Required checks

- one-column card stack
- 44px tap target guard
- compact persona stories
- visible warning badges
- no horizontal overflow
- sticky action region available for future route actions

## Phase 31 note

The mobile deck uses separate CSS so small screens get their own hierarchy while keeping the same visual grammar as desktop.
