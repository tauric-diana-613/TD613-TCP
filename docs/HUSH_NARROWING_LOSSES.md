# Hush Narrowing Losses

Phase 30 turns epistemic loss into a measurable cockpit surface.

## Loss channels

- `payload`: facts or claim payload weakened.
- `literal`: protected literal dropped or changed.
- `event`: event shape, timing, version, copy, or actor relation damaged.
- `register`: source register erased in preserve mode.
- `target`: target register invisible or overcooked.
- `export`: receipt omits custody layer or stores private text by default.
- `ui`: report truth fails to appear in the product route.
- `docs`: capability exists in code/tests but not operator documentation.

## Route state

Hard losses route to `hold`. Reviewable losses route to `warning`. Clean receipts route to `receipt-ready`.
