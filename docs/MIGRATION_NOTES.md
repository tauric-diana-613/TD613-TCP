# MIGRATION NOTES

This file tracks the architectural turns that matter for reading the current repo correctly.

## Prototype to engine

The original browser prototype proved three things worth preserving:

1. stylometry could be made legible in public
2. branch / wave / harbor theory could be staged without hiding the math
3. persona and archive artifacts were useful enough to deserve portability

The current repo keeps those ideas, but moves them into maintained modules, tests, schemas, reports, and browser/runtime parity checks.

## Retrieval-first turn

The first major migration was away from a style-only playground and toward a retrieval-first cadence lab. That turn added:

- semantic audits
- protected-anchor integrity
- candidate and fixture proofs
- swap-matrix and false-neighbor diagnostics
- mask and trainer retrieval contracts

That was the point where TCP stopped pretending that visible style shift alone was enough.

## Anti-PRCS-A turn

The next migration was doctrinal rather than cosmetic. `TD613 Aperture` was moved out of the role of quiet admissibility sorter and into the role of counter-tool:

- warnings stay visible
- repair stays visible
- catastrophic faults can block
- ordinary misses do not silently reroute to source

This is where explicit hold dockets became part of the contract instead of a UI afterthought.

## Generator V2 turn

The current migration is the most important engine turn so far.

`buildCadenceTransfer()` is still the public writer API, but Generator V2 now sits behind it by default. That means:

- V2 writes from source text directly
- V2 does not seed from legacy-authored text
- legacy remains available only through explicit compatibility exports
- candidate selection and holds are first-class
- `generatorVersion`, `generationDocket`, `candidateLedger`, and `holdStatus` are part of the maintained return shape

In plain language: the old engine is no longer supposed to win by default just because it is safer or flatter.

## Diagnostics turn

The diagnostics battery used to tell the truth about retrieval, swap, mask, and trainer behavior while only indirectly witnessing the writer. The current repo now adds a first-class generator audit so the diagnostics surface can answer:

- did V2 actually author this lane?
- did the lane land or hold?
- was the landed result structurally live?
- did it stay semantically bounded?
- were anchors preserved?

That audit is generated by `node scripts/run-diagnostics-battery.mjs` and written into `reports/diagnostics/`.

## What is still intentionally stable

Several public contracts did not change during this migration:

- `buildCadenceTransfer()` stays the main API
- browser parity stays required
- Persona/Homebase, Deck, and Trainer still consume one shared engine truth
- Safe Harbor and Aperture proper remain downstream consumers rather than alternate hidden writers

The migration goal is not to make TCP feel like a new product every patch. The goal is to keep the same public surface while making the maintained writer, audit, and diagnostics contracts much less fragile.
