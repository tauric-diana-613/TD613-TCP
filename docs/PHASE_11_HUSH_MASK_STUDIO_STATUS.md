# Phase 11 Status — Adversarial Bench → Hush Mask Studio

## Why Phase 11 exists

Phase 11 exists because the useful Hush workflow already lives at `/adversarial-bench`, but the visible language and mask path still carried old bench and Persona-room residue.

Hush needs to be the clean mask studio inside The Cadence Playground: choose a mask, build a mask, write the message, swap the surface, review the pressure, and export only after warnings are visible.

## Why Hush stays on /adversarial-bench

`/adversarial-bench` already contains the serious local stylometry pipeline: Escape Vector, Ingestion Friction, Controller, Mask Memory, Recognition Field, Flight Recorder, Claim Ladder, and Report Export.

Phase 11 renames and upgrades that page rather than creating a second competing app.

## What Hush replaces conceptually

Hush replaces the need to move through Homebase, Deck, or Clone for the basic whistleblower-facing mask path.

It gives one path:

1. select a Hush mask;
2. create a custom mask when needed;
3. watch the profile refine;
4. write the message;
5. swap the surface;
6. review pressure;
7. export only what should travel.

## What Hush does not delete yet

Homebase, Deck, and Clone remain available as legacy and lab chambers. They are not design parents for Hush.

Phase 11 treats Homebase, Deck, and Clone as salvage yards, not design parents.

## New Hush mask library

Phase 11 adds a Hush-native mask library in `app/data/hush-masks.js`. These masks are not legacy Personas. They are local stylometric targets with descriptions, intended use, risk tells, sample seeds, transform hints, and pressure warnings.

## Custom mask workflow

The Customize tab lets an operator add writing samples to a custom mask. Each sample updates the composite profile. Default exports exclude raw sample text.

## Swap workflow

The Swap Surface button generates local candidates, scores them, and selects the best candidate by mask match, semantic fidelity, protected literal preservation, source reduction, and context pressure.

## Profile matching

Profile matching uses local tolerance-band convergence. A high mask match with low semantic fidelity is treated as a failed swap.

## Ontology quarantine

Phase 11 does not re-enable legacy ontology gating. Ontology may classify, annotate, warn, and route. Ontology may not rewrite, decide, override metrics, or claim meaning beyond measured evidence.

## Clone salvage

Clone contributes the useful pattern: corpus input, profile extraction, validation, export. The Clone chamber does not supply the Hush design.

## Whistleblower posture

Hush is designed for pressure review before exposure. It does not claim external recognition outcomes. It gives the operator a clearer local surface before the message travels.

## Next work

Future work can tune mask recipes, improve candidate generation, add import/export UI for custom masks, and expand fixture coverage for Hush-specific swap behavior.
