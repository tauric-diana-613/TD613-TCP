# Phase 20 Status — Hush Alien Console

Phase 20 redesigns Hush as a futurecore-goth operator console while preserving the Phase 19 transformation engine.

This phase is a surface and navigation overhaul. It makes the main operator path visible first: message, mask, transform, heat, copy. Advanced custody machinery remains available through Vault and Lab drawers, but it no longer competes with the primary transform workflow.

## Surface doctrine

A powerful tool with a confusing surface remains a barrier.
A beautiful interface that hides warnings becomes a trap.
A futurecore-goth instrument must still tell the truth.
A whistleblower under pressure needs a path, not a maze.

## Visual posture

The Alien Console layer uses black glass, violet/cyan signal light, bone text, pressure tiles, mask route cards, and recessed Vault/Lab machinery. It is designed to feel like a found instrument rather than a generic dashboard.

## Boundary

Phase 20 does not rewrite the engine. It does not change release policy, scoring, claim integrity, source-residue logic, or syntax recomposition.

It does not claim anonymity, untraceability, platform-proof behavior, publication safety, or identity outcomes.

## Acceptance pressure

Phase 20 passes only if:

- the main operator path is obvious before Vault or Lab are opened;
- the message input remains the primary first field;
- mask selection can happen through route cards while the dropdown remains available;
- Transform remains the central action;
- Phase 19 heat metrics are visible as readable operator pressure;
- Vault and Lab remain accessible but recessed;
- reduced-motion users are respected;
- existing Hush engine tests continue to pass.
