# Phase 11 — Action Gates

Status: implementation doctrine.

Phase 11 action gates prevent UI controls from outrunning packet law.

## Doctrine

A button is a petition.

Every visible action must be evaluated by `evaluateHushPhase11Action()` before the drawer treats it as available, review-required, blocked, or not applicable.

## Gate statuses

- `available` — action may proceed under current packet posture
- `review-required` — action may help custody but requires operator awareness
- `blocked` — action conflicts with evidence, release discipline, boundary posture, or export law
- `not-applicable` — action has no jurisdiction over the current packet state

## Required explanations

Every blocked or review-required action must expose:

- source phase
- reason
- repair route
- blocking fields where applicable

No silent gray buttons. No implied authority.

## Export distinction

Redacted export is release speech. Private backup is custody speech.

A blocked packet may still allow dashboard-summary copy if the summary preserves blockers and non-claims. Repair speech is different from release speech.

## Forbidden UI moves

- mark release-candidate without Phase 10 support
- mark sealed without boundary completion
- export raw surfaces through redacted export
- present fixture-backed evidence as live evidence
- treat provider output as self-certified compliance
- treat Safe Harbor, Aperture, or EO-RFD as a validator bypass
