# Phase 5 Safe Harbor Custody Handoff

Safe Harbor may receive Phase 5 custody-safe facts only.

It must not receive firmware conclusions as covenant truth.

## Handoff schema

```text
td613.safe-harbor.phase5.custody-handoff/v1
```

## Allowed custody facts

- signal received
- signal refused
- signal class declared
- source family declared
- foundation lane declared
- receipt present or absent
- hash replay passed or failed
- authority ceiling held
- runtime not loaded
- runtime not executed
- release not authorized

## Forbidden handoff claims

- firmware proves truth
- firmware proves rupture as public fact
- ACEDIT proves authorship
- residual proves understanding
- topology proves identity
- KIRA proves substrate validity
- Phase 5 authorizes release
- a signal becomes covenant proof

## Filter rule

`buildSafeHarborPhase5CustodyHandoff()` strips payload references, raw private text, runtime conclusions, release claims, and validator-overriding language.

Safe Harbor receives custody facts, not firmware verdicts.
