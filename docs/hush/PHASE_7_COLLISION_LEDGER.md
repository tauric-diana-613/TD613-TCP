# Phase 7 Collision Ledger

The collision ledger records overlap instead of hiding it.

Collision types include:

- duplicate mask ID
- duplicate label
- role overlap
- source overlap
- retired near-duplicate
- target-register overlap
- unresolved collision

The registry records collisions while preserving all source records.

No canonical mask is silently overwritten.

No extension mask is silently promoted.

The collision ledger is an accession-control surface, not a deletion tool.
