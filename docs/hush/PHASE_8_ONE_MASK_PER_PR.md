# Phase 8 One-Mask-Per-PR Rule

Phase 8 proceeds one mask at a time.

A valid mask PR must touch one mask packet only.

It must include:

- mask ID
- label
- source file
- source index
- registry record hash
- collision posture
- calibration summary
- human imperfection ledger
- anti-slop result
- sample export false
- public-default false
- packet hash and replay
- tests run

No multi-mask PRs.

No gallery refactors hidden inside a mask packet PR.

No extension mask promotion inside canonical packetization.

The room stays small enough to see what changed.
