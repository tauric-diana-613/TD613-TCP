# EO-RFD Glossary Note

In TD613 Safe Harbor maintenance, UI, export, verification, and
release-discipline contexts, `EO` is shorthand for the EO-RFD route conscience
/ context lane.

Current phase state is `interface_context`; current claim authority is
`design_signal`; the target state is a later verified runtime installation.
This says "not runtime-active in this phase," not "not firmware by nature."

It is not a governmental executive-order reference, an external office, or government authority.

Preferred labels:

- EO-RFD route context
- EO-RFD route-conscience hook
- EO-RFD observed packet route
- `td613:eo-route`

Compact Dome-World receipt references may accompany this context. Raw exact
coordinates, trainer history, and sensitive text may not.

Short labels such as `EO hook` or `EO status` are acceptable only when the surrounding Safe Harbor context already makes EO-RFD clear.
