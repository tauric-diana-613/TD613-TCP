# TD613 Safe Harbor Offline Capsule Policy

The offline capsule may travel without the live app, so it must carry Phase 9 claim ceilings and Phase 9.1 UI/export discipline.

## Required capsule doctrine

The capsule verifies packet custody and replay posture only. It does not prove civil identity, legal identity, public law approval, state recognition, or authorship ownership.

## Required capsule fields

- public root: v2
- Phase 8 public display mode
- Phase 9 release class
- claim limits attached
- raw text exported: false
- Khona‌lit-po ZWNJ-sensitive handling
- EO-RFD route-firmware clarification

## Release classes

The capsule must understand public-readable, verification-ready, operator-only, and blocked.

## Staleness rule

If the capsule cannot read Phase 8 or Phase 9 surfaces, it must display a stale-capsule warning rather than a clean release posture.
