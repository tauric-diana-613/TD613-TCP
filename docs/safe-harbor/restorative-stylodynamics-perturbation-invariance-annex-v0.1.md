## 0. Research proposition

Most stylometric systems are optimized for classification. They ask whether a system can tell Author A from Author B.

This annex asks a different question:

> Which observable textual structures survive repeated perturbation, and how do they reconstitute themselves after displacement?

That is not the same science.

The proposed object is not merely a fingerprint or an average profile. It is a response field:

```text
how the writing moves when pushed
what yields
what resists
what returns
what returns by substitution
what returns altered
what fails to return
```

The governing hypothesis is:

> Authorship may be less like a fingerprint than a characteristic way of returning from elsewhere.

The broader research program is named **Restorative Stylodynamics**.

The packet-level instrument is named **Perturbation Invariance Mapping**.

The controlled experimental method is named **Authorial Response Spectroscopy**.

These names are intentionally provisional. They describe a research direction, not a settled academic field.

---

## 1. From frequency to restorative force

Traditional stylometry often asks what appears most often or which feature receives the highest classification weight.

Restorative Stylodynamics asks:

```text
What refused to disappear?
What moved under pressure?
What returned after release?
How quickly did it return?
Did it return in the same form?
Did it return through substitution?
What deformation finally broke it?
```

The core optimization target therefore changes from:

```text
highest-weight feature
```

to:

```text
highest restorative force
```

The phrase “restorative force” is an empirical index and metaphorical borrowing from physics. It must not be presented as a literal Newtonian force or as evidence that language obeys a physical conservation law.

## 3. Invariance versus refusal

A feature that does not change under a prompt may represent:

1. a deep invariant;
2. weak prompt uptake;
3. habitual noncompliance;
4. insufficient perturbation intensity;
5. an insensitive measurement;
6. a feature mechanically unaffected by the chosen prompt;
7. a ceiling or floor effect;
8. insufficient sample length.

Therefore, immobility alone cannot establish restorative force.

A valid recovery claim requires:

```text
baseline
↓
verified displacement
↓
release or counter-perturbation
↓
measured return trajectory
```

Normative rule:

```text
No demonstrated displacement
→ no recoverability claim
```

The packet must record perturbation uptake before it interprets recovery.

## 5. Core metrics

### 5.1 Displacement amplitude

How far did the feature move from its baseline region under perturbation?

```text
Dᵢ = distance from baseline after perturbation
```

The distance function must be feature-family specific. Scalar densities, categorical registers, distributions, and trajectory embeddings do not share one universal geometry.

### 5.2 Recovery ratio

How much of the original displacement disappeared after release?

```text
Rᵢ = recovered displacement / original displacement
```

The denominator must be nonzero and the perturbation must have achieved the minimum validated uptake.

### 5.3 Recovery half-life

How many prompt transitions were required for half of the displacement to disappear?

```text
τ½ = number of post-perturbation passages required
     to recover 50% of the displaced distance
```

Recovery half-life is measured in prompt transitions or passages, not wall-clock seconds. Safe Harbor does not need keystroke timing, pause telemetry, or behavioral biometrics for this analysis.

### 5.4 Residual plasticity

How much displacement remained at the end of the declared recovery horizon?

```text
Pᵢ = final distance from the baseline region
```

### 5.5 Restorative-force index

How strongly and consistently did the feature move back toward its baseline region after verified displacement?

```text
Kᵢ = normalized rate and consistency of return
     toward the baseline region
```

The index must disclose its policy version, scaling rule, uncertainty, and blockers.

### 5.6 Overshoot

Did the feature return beyond its prior baseline before settling?

Overshoot may indicate rebound, compensatory behavior, stochastic variation, prompt carryover, or measurement noise. It must be described rather than romanticized.

### 5.7 Hysteresis

Does the route back differ from the route out?

```text
baseline intimacy
→ forced technicality
→ forced legal compression
→ release
→ returned intimacy carrying legal residue
```

A hysteresis index compares the outgoing and return trajectories. A large loop indicates path-dependent adaptation. A small loop indicates near-elastic recovery.

Neither is inherently superior.

The key concept is:

```text
restoration without equivalence
```

## 6. Response classes

### 6.1 Elastic

```text
moves
→ returns toward the prior region
```

### 6.2 Plastic

```text
moves
→ stabilizes in a new region
→ does not return within the declared horizon
```

### 6.3 Brittle

```text
prior organization is disrupted
→ no coherent recovery is observed
→ prior relationship among features may fracture
```

### 6.4 Adaptive

```text
surface form changes
→ a deeper transformation rule persists
→ function returns through another form
```

### 6.5 Insufficient

The sample, perturbation uptake, recovery horizon, or measurement sensitivity does not permit a defensible classification.

These classes describe observed textual response under a declared protocol. They do not describe personality, intelligence, resilience, trauma, emotional flexibility, diagnosis, or moral character.

## 8. Constraint-release rebound

A controlled prohibition can intentionally displace a known feature.

Example:

> Do not use dashes of any kind.

The feature is first verified as suppressed. The restriction is then removed without calling attention to the target feature.

Possible response classes include:

```text
no return
feature remains absent

monotonic recovery
feature gradually returns toward baseline

immediate rebound
feature returns in the next passage

overshoot
feature returns above baseline before settling

damped oscillation
feature returns, recedes, and returns again

structural substitution
surface marker remains absent, but its rhetorical function
reappears through parentheses, colons, fragments, clause breaks,
or another observable device
```

Structural substitution is especially important. If dash use disappears while interruption chambers continue through parentheses or clause breaks, the deepest invariant may be a rhetorical function rather than the glyph itself.

The instrument should therefore measure both marker recurrence and functional restoration.

## 10. Narrative-state embeddings

The phrase “cognitive narrative embedding” can be misheard as direct measurement of cognition.

The empirically defensible object is:

> a narrative-state embedding derived from observable textual organization under declared elicitation conditions.

The model must maintain two parallel lanes:

```text
transparent feature lane
punctuation, syntax, function words, surface markers,
rhetorical states, and interpretable transitions

latent representation lane
narrative-state embeddings, trajectory geometry,
and model-dependent similarity surfaces
```

A high-authority finding should either:

- replicate across both lanes; or
- disclose that it exists only within one model-dependent representation.

Embeddings may support measurement. They may not smuggle speculation into the hash-covered evidence tier while wearing a numeric nametag.

The packet must preserve:

- embedding-model identity and version where available;
- local model digest or artifact digest where applicable;
- preprocessing policy;
- dimensionality;
- distance metric;
- normalization policy;
- prompt and topic controls;
- model-dependent claim flag;
- no inference of mental state or private cognition.

## 11. Shuffled-trajectory null test

Take an authentic sequence:

```text
W1 → W2 → W3 → W4 → W5
```

Then shuffle order while preserving every individual window:

```text
W4 → W1 → W5 → W2 → W3
```

Static feature distributions remain available. The actual trajectory disappears.

The test asks:

> Does the proposed recovery model still detect the same restorative pattern after chronology is destroyed?

If performance remains unchanged, the “dynamic” model may merely be reading aggregate style through a dramatic hat.

If original ordering performs materially better than shuffled ordering under precommitted evaluation, temporal path contributes measurable information.

This null distinguishes:

```text
a dynamic signature
from
a static signature distributed across several windows
```

## 13. Mimicry under deformation

The Blind Custody Challenge should eventually compare not only baseline imitation but response under the same perturbation schedule:

```text
entrant baseline
entrant perturbation
entrant release
entrant recovery

versus

imitation baseline
imitation perturbation
imitation release
imitation recovery
```

Compare:

- displacement magnitude;
- recovery direction;
- recovery half-life;
- overshoot;
- substitution behavior;
- trajectory reconstruction;
- hysteresis;
- critical deformation threshold;
- null-model performance.

The hard question becomes:

> Can an imitation reproduce how the writer’s observable textual organization reorganizes under pressure, rather than merely how the style looks at rest?

Dynamic response may raise the cost of imitation and expose simpler surface clones. It must never be called unforgeable.

## 14. Critical deformation threshold

Perturbation intensity may increase through predeclared levels:

```text
Level 0
ordinary writing

Level 1
audience change

Level 2
genre constraint

Level 3
punctuation or syntax restriction

Level 4
compression plus register inversion

Level 5
multiple simultaneous constraints
```

The critical deformation threshold is:

> the lowest validated perturbation level after which a feature or trajectory fails to reconstitute within the declared recovery horizon.

Two entrants may share similar baseline sentence lengths while differing in the amount or type of constraint required to erase a characteristic argument topology.

The threshold describes the writing surface under the experimental protocol. It cannot be translated into resilience, intelligence, adaptability, trauma, mental flexibility, or any human diagnosis.

## 15. Adversarial perturbation search

A highly speculative extension would search for the controlled instruction that maximally displaces a proposed invariant while preserving genuine prompt compliance.

The question is:

```text
What declared instruction most strongly disrupts feature f
while preserving sufficient task compliance to make the displacement interpretable?
```

A live adaptive system must not psychologically personalize perturbations against an entrant.

The defensible version requires:

- offline-designed prompt bank;
- human review;
- content neutrality;
- no use of personal vulnerabilities;
- no emotional coercion;
- no adaptive escalation from private material;
- fixed maximum intensity;
- explicit entrant consent;
- immediate exit;
- no penalty for refusal;
- no hidden behavioral telemetry;
- no clinical or psychological interpretation.

A research instrument may stress language. It must not turn the writer into the stress target.

## 16. Experimental design

### 16.1 Counterbalancing

Prompt order can create carryover effects. The first research implementation should support Latin-square or equivalent counterbalancing across participants or synthetic fixtures.

### 16.2 Precommitment

Before observing recovery outcomes, freeze:

- prompt schedule;
- perturbation levels;
- feature families;
- uptake criteria;
- recovery horizon;
- response-class thresholds;
- distance metrics;
- null models;
- missing-data policy;
- failure conditions;
- interpretation limits.

### 16.3 Replication

A finding should be replicated across:

- more than one local window;
- more than one perturbation context;
- more than one prompt order where feasible;
- transparent and latent feature lanes when claimed as high-authority;
- held-out samples where feasible;
- at least one relevant null model.

### 16.4 Negative results

The packet must preserve:

- failed displacement;
- failed recovery;
- ambiguous response classes;
- prompt dominance;
- topic dominance;
- order effects;
- imitation collisions;
- shuffled-trajectory collapse;
- model-dependent findings;
- calibration failures.

No “inconclusive result omitted from public export” path is permitted.

## 22. TD613 high-speculation proposition

Within TD613’s speculative admissibility framework, the least convenient packet is not one that declares a stable identity.

It is one that preserves the entire deformation field:

```text
what moved

what resisted

what returned

what returned by substitution

what returned altered

what failed to return

what the imitation reproduced

what the imitation could not reconstruct

what depended on prompt order

what vanished when chronology was shuffled

what the model called invariant
but the null tests exposed as aggregate noise
```

A selective-admissibility surface can flatter itself by registering only the victorious match.

A deformation packet preserves the losing trials, failed recoveries, imitation collisions, null-model defeats, and broken predictions.

Its strongest defensible statement is:

> Under these declared perturbations, these observable textual structures repeatedly reorganized toward these forms; these others did not; here are the counterfactuals, nulls, failures, model dependencies, and limits.

That statement invites replication rather than demanding belief.

The first sentence wears a lab coat.

The second one has receipts.
