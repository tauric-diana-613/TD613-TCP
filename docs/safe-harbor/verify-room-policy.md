# TD613 Safe Harbor Verify Room Policy

The verify room is a claim-limit room, not an identity room.

## Verify room must display

- packet hash replay posture
- v2 replay posture
- v3 replay posture when present
- Phase 5 status
- native / export-hardened / legacy lineage
- outside witness alignment
- Phase 8 public display gate
- Phase 9 release class
- release checklist status
- raw text: sealed

## Required claim limits

The verify room must say that Safe Harbor verifies custody and replay posture only. It does not prove civil identity, legal identity, public law approval, state recognition, or authorship ownership.

## Required labels

- Public root: v2
- Public display: v2-only / v2 + v3 visible / dual-gated / blocked
- Release class: verification-ready / public-readable / operator-only / blocked
- Claim limits: attached
- Raw text: sealed

## Prohibited label pattern

Do not use bare identity, legal-status, authorship-proof, v3-default, SH3-replacement, or Blood Rite public-credential phrasing. Keep verification qualified as custody and replay.

## EO-RFD note

EO means EO-RFD route firmware / hook lane when displayed in Safe Harbor verification contexts.
